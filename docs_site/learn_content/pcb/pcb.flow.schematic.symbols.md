A schematic is the logical description of a circuit: it captures *what connects to what*, deliberately divorced from the physical geometry of the board. Its atomic unit is the **symbol**, an abstract graphic that stands in for a physical component (a resistor, an op-amp, a connector, a microcontroller). A symbol is not a picture of the part; it is a standardized glyph plus a set of **pins**, each carrying a name, a number that maps to a package terminal, and an electrical type. Standardized graphic symbols exist precisely so that a schematic is unambiguous across authors and tools: the two dominant catalogs are the IEC 60617 series and its longstanding IEEE counterpart, which fix the appearance of resistors, capacitors, logic gates, transistors, and the like.

A symbol on a drawing is more precisely a **component instance** with a reference designator (for example `R12`, `U3`), a value or part number, and the pin-to-net attachments that give it meaning. The same library symbol may be placed many times; each placement is a distinct instance with its own designator. Multi-unit parts (a quad op-amp, a hex inverter) split one physical package across several drawable **gates or units** that can be scattered across the schematic, sharing one designator and one set of shared power pins.

## Sheets and multi-sheet designs

Real designs rarely fit on one page. A schematic is therefore organized into **sheets**, each a bounded drawing surface with a title block recording revision, author, and sheet number. Connectivity crosses sheet boundaries through **off-page connectors** (labeled ports that declare "this wire on sheet 2 is the same net as this wire on sheet 5") and through **named nets and buses**, where a shared label rather than a drawn wire establishes the connection. Buses bundle many related signals (`DATA[0..7]`, `ADDR[0..15]`) into a single drawn line to reduce clutter, expanding into their member nets at the endpoints.

## Hierarchical design

Beyond flat multi-sheet drawings, large systems use **hierarchy**: a sheet is represented on its parent by a single block with **hierarchical pins (ports)**, exactly as a function is called by its signature rather than its body. A power-supply subsheet, for instance, appears upstream as one block exposing `VIN`, `GND`, and `+3V3`. Hierarchy enables *reuse* (the same subsheet instantiated multiple times, each instance a separate copy of the underlying nets) and *encapsulation* (the parent need not know the internal wiring). When the design is flattened for downstream use, the tool composes the hierarchy into a single global connectivity model, uniquifying repeated blocks so that two instances of the same channel do not accidentally short together. The governing discipline is that the drawn hierarchy and the extracted netlist must be provably equivalent, so the abstraction adds clarity without changing electrical meaning.
