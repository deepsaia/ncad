## Components as instanced occurrences

Each placed part on the board has an associated 3D body -- typically an exchanged solid model -- and a placement in the layout: a location \((p_x,p_y)\), a rotation \(\theta\) about the board normal, and a side (top or bottom). Lowering the assembly means realizing every placement as an occurrence of a shared prototype body, positioned by a rigid transform. Rather than copying geometry per instance, the assembly is stored as a label tree: each unique body is a prototype stored once, and every placement is a lightweight reference that carries only a location and attributes such as name, color, and reference designator. This is the eXtended framework (XCAF) shape model, and it mirrors the product structure of the STEP standard, where an assembly is a graph of next-assembly-usage occurrences and mapped items, each holding an axis placement.

Instancing is essential at board scale. A design may place hundreds of identical decoupling capacitors; storing one prototype and \(N\) transforms keeps memory and file size proportional to the number of *distinct* parts, not the number of placements. It also preserves identity: editing the prototype updates every occurrence at once, and each occurrence retains a stable handle for selection, coloring, and downstream interference reporting.

## The placement transform

The placement transform composes several rigid motions. Writing rotations and translations as homogeneous \(4\times 4\) matrices,
\[ T = T_{\text{trans}}(p_x, p_y, z_s)\; R_z(\theta)\; M_{\text{side}}\; T_{\text{seat}}, \]
where \(T_{\text{seat}}\) aligns the model's local origin and seating plane to the footprint's insertion point and the board surface, \(M_{\text{side}}\) selects top-versus-bottom mounting, and \(R_z(\theta)\) with \(T_{\text{trans}}\) applies the layout's in-plane rotation and position. Because occurrences compose, a board placed inside an enclosure simply inherits the enclosure's transform on top of its own -- the same matrix algebra all the way up the tree.

Bottom-side placement is where sign conventions bite. Mounting a part on the far face of the board is a reflection: an orientation-reversing map with \(\det M_{\text{side}} = -1\) (for example a 180-degree turn about an in-plane axis) followed by a translation to the bottom surface. If the reflection is applied inconsistently with the model's local frame, the body ends up buried in the board or mirrored -- one of the most common ECAD-to-MCAD defects. Because a reflection changes handedness, any text, polarity marks, or asymmetric bodies on a flipped part must be checked, not assumed correct.

A second recurring pitfall is the model's local origin. Exchanged component solids do not agree on where their origin sits -- body center, pin 1, or a mounting datum -- so \(T_{\text{seat}}\) must map whatever convention the model uses onto the footprint's insertion point and seating height, including any standoff. Getting that mapping wrong shifts every instance of the part by a fixed offset. The saving grace is that the alignment is per-prototype: fixing it once corrects all occurrences, which is exactly the leverage the instanced sub-assembly structure provides. This is why keeping the exchange keyed to the footprint or part number (so the right prototype and the right seating datum travel together) is the crux of reliable ECAD-MCAD collaboration.
