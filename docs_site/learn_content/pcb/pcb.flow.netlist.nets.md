A **net** is the fundamental unit of electrical connectivity: the set of all pins and pads that are meant to be at the same electrical potential, that is, joined by conductor. Every wire drawn on a schematic, every named label, and every bus member ultimately resolves to a net. The complete list of nets, together with the pins each contains, is the **netlist**, the formal handoff artifact from the logical schematic domain to the physical layout domain. A netlist is therefore best understood as a partition of the design's pins into equivalence classes, one class per net, usually presented as a numbered table (`net 1: GND -> U1.7, C3.2, J1.4, ...`) with a stable identifier so the connection can be traced end to end through fabrication and bare-board test.

Formally the connectivity is a graph. Let each pin be a vertex; a net of \(k\) pins asserts that all \(k\) vertices belong to one connected component. The layout tool's job is to realize every such component in copper. Before routing exists, though, the required connections must be *shown*, and that is the role of the **ratsnest**: for each net the tool draws thin straight "air wires" between its pads, so the designer sees exactly which pads still owe a connection. Ratsnest lines are not copper; they are a live to-do list that shrinks as tracks are routed and disappears when a net is fully connected.

## Why a spanning tree

A net of \(k\) pads needs only \(k-1\) conductor links to be electrically complete, not the \(\binom{k}{2}\) links of every-pin-to-every-pin. To display the *least misleading* set of air wires, ratsnest generators compute a **minimum spanning tree** over the net's pads, minimizing total displayed length

\[ W = \min_{T \in \mathcal{T}} \sum_{(u,v)\in T} d(u,v), \qquad d(u,v)=\lVert p_u - p_v \rVert, \]

where \(\mathcal{T}\) is the set of spanning trees on the net's pads and \(d\) is (typically) Euclidean distance. The MST both keeps the display uncluttered and hints at an efficient routing topology. As pads move, the tree is recomputed, so the ratsnest always reflects the shortest current set of pending connections.

Connectivity is also the anchor for downstream integrity. The netlist drives **DRC connectivity checks** (copper that touches pads of two *different* nets is a short; a net whose pads are not all joined by copper is an *open* / unrouted connection), and it is exported in standardized machine forms for bare-board electrical test so a fabricator can verify the finished board's nets match the design. Because the net table is the shared vocabulary of schematic, layout, simulation, and test, keeping net names and numbers stable and meaningful is not cosmetic; it is what lets a signal be reasoned about consistently across every stage of the flow.
