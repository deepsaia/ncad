Holes are the drilled features of a board, and the model distinguishes two kinds by their electrical role. A **plated through hole (PTH)** has a copper-plated barrel that electrically connects the layers it passes through; it serves both as a via and as the receptacle for through-hole component leads. A **non-plated through hole (NPTH)** has no copper barrel and is purely mechanical: mounting holes, tooling and registration holes, and clearance holes for hardware. The plating attribute changes not only fabrication (plated holes are metallized before the outer-layer image) but also the clearance and annular-ring rules that apply.

A crucial subtlety is the difference between the **drill size** and the **finished hole size**. Plating deposits copper on the barrel wall, shrinking the hole, so the fabricator drills oversize:

\[ d_{\text{drill}} \;\approx\; d_{\text{finished}} + 2\,t_{\text{plating}} \;(+\ \text{process allowance}). \]

The data model should carry the *finished* diameter as the design intent and let plating thickness derive the tool size, because the finished dimension is what a lead or screw must fit. Tolerance bands attach to the finished size, and they interact with the annular-ring rule on any pad that surrounds the hole.

Hole **aspect ratio**, the board thickness divided by the hole diameter, governs how well plating chemistry can throw copper down a deep, narrow barrel; fabricators cap it (commonly around 8:1 to 10:1) and it directly limits the smallest plated hole allowed in a thick board. Registration, the alignment of the drilled hole to the copper pattern, sets how much annular ring the pad must reserve so that drill wander does not break out. Both are why the hole is not an isolated feature but one tied to stackup thickness and pad geometry.

Aggregated across the whole board, holes form the **drill table**: a summary that groups every hole by finished diameter, plating status, and tolerance, and records the count (number of "hits") per group. This table is the human-readable side of the machine **NC drill program** (the de facto Excellon format), which lists tool definitions and the coordinate list for each tool. The number of distinct tools drives tool changes and therefore cost and time, so consolidating nearby sizes to a common drill is a real DFM lever the table makes visible. Slots, castellated (edge-plated) holes, and counterbores are variants the same table must represent.

In the data model, individual holes reference the padstack/drill definitions that produced them, while the drill table is a **derived roll-up** grouped and sorted deterministically so that the same board always emits the same tool list and coordinates. Treating holes as first-class features with finished-size intent, plating status, and tolerance, and the drill table as their deterministic aggregation, is what lets fabrication output, aspect-ratio checks, and annular-ring verification all draw from one consistent source.
