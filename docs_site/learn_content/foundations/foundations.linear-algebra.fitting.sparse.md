A **sparse linear system** is a system \(A x = b\) in which the matrix \(A\) is overwhelmingly zero, typically with only a handful of nonzeros per row regardless of how large the matrix is. Such systems are the norm rather than the exception in engineering because they arise whenever a continuous problem is discretized: finite-element, finite-difference, and finite-volume models couple each node only to its immediate neighbors, so a mesh with millions of unknowns yields a matrix that is enormous but almost empty. Constraint systems, network and circuit models, and graph Laplacians produce the same structure. Exploiting sparsity is not an optimization but a necessity; treating the matrix as dense would demand \(O(n^2)\) storage and \(O(n^3)\) work that is impossible at realistic scale.

## Storage and structure

Sparse matrices are stored in formats that keep only the nonzeros and their indices, such as compressed sparse row or column (CSR/CSC) and coordinate (COO) forms. The pattern of nonzeros, its **sparsity structure**, is as important as the numbers, because it determines how much extra work a solver will create. Many engineering matrices are also symmetric positive definite (the stiffness matrix of a well-posed static analysis is a canonical example), which unlocks the most efficient and reliable algorithms.

## Direct versus iterative methods

**Direct methods** factor the matrix, using sparse Cholesky for symmetric positive definite systems or sparse LU otherwise. The central difficulty is **fill-in**: factorization creates new nonzeros in positions that were originally zero, and unchecked fill can consume all the memory the sparsity was meant to save. Solvers therefore apply fill-reducing reorderings such as minimum-degree or nested-dissection permutations before factoring. Direct methods are robust and predictable and let one reuse a single factorization for many right-hand sides, at the cost of high memory use on very large three-dimensional problems.

**Iterative methods** never form a factorization; they improve an approximate solution using only matrix-vector products, which cost proportionally to the number of nonzeros. Krylov-subspace methods dominate this class: the conjugate gradient (CG) method for symmetric positive definite systems, and GMRES or BiCGSTAB for general ones. Their convergence depends on the spectrum of \(A\), so they are paired with a **preconditioner** \(M \approx A\) that clusters the eigenvalues and accelerates convergence, common choices being incomplete factorizations (ILU) and algebraic multigrid. Iterative methods scale to the largest problems with modest memory, but their success hinges on finding an effective preconditioner. The practical decision between the two families weighs robustness and reuse (direct) against memory footprint and scalability (iterative), and large simulation codes routinely offer both.
