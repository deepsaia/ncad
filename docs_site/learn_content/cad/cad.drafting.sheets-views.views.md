## Multiview and pictorial projection

A drawing view is a two-dimensional projection of a three-dimensional model. Engineering drawings use parallel (orthographic) projection, in which the projectors are perpendicular to the image plane, so parallel model edges stay parallel and dimensions on faces parallel to the plane are undistorted. A base (principal) view is placed first; projected views (top, front, side) are generated by projecting onto the other principal planes and are kept aligned so that a point in one view lines up horizontally or vertically with the same point in an adjacent view. The layout follows a declared convention: third-angle projection places each view on the side it is seen from, first-angle on the opposite side. The two are mirror arrangements, and the sheet states which is used with a truncated-cone symbol.

Orthographic projection onto, for example, the XY plane simply drops the depth coordinate:

\[ P(x,y,z) = (x,\,y). \]

Isometric and other pictorial (axonometric) views instead project along a direction equally inclined to all three model axes, so the three principal axes appear 120 degrees apart and edges along them share a single foreshortening factor. True isometric foreshortening is \(\sqrt{2/3}\approx 0.816\), although pictorials are commonly drawn full-scale along the axes for convenience. An auxiliary view projects onto a plane set parallel to an inclined face so that face shows its true shape and size; it is the standard way to dimension features that are not parallel to any principal plane.

## Hidden-line removal

A raw projection overlaps every edge, so a readable drawing must determine which edges are visible and which are occluded, drawing the former as solid lines and the latter as dashed hidden lines (or omitting them). This is hidden-line removal (HLR). Exact, or precise, HLR computes visibility analytically on the boundary-representation edges and curves and produces crisp vector output suitable for dimensioning; approximate HLR works from a tessellation and is faster but coarser.

A classic exact method assigns each edge a quantitative invisibility: the count of front-facing faces that lie between the edge and the eye. An edge is visible only where that count is zero. The count changes by \(\pm 1\) only where the edge visually crosses a silhouette (contour) edge, so the algorithm locates the silhouette curves, evaluates the invisibility once per edge segment, and propagates it across crossings instead of testing every point against every face. Because views are computed from the model, they are associative: when the model rebuilds, the projected edges, silhouettes, and hidden-line classification regenerate deterministically.
