A **setup** captures one physical staging of the job: how the part is fixtured, which way is up, where the program origin sits, and which face(s) the machine can reach in that clamping. A **work coordinate system** (WCS, also called the work offset or program zero) is the frame in which all programmed coordinates are expressed. Separating the WCS from the machine coordinate system (MCS) is what lets the same part program run regardless of exactly where the vise sits on the table: the operator probes or sets the work offset once, and the controller adds it to every move.

The axis and motion conventions are standardized so that a program is portable and unambiguous. The controlling standard defines a right-handed Cartesian frame in which the **Z axis is parallel to the principal spindle**, positive Z pointing in the direction that *increases* the distance between tool and workpiece (i.e., retract). The X axis is horizontal and, on a vertical machine, chosen as the longer table travel; Y completes the right-handed set. Rotary axes A, B, C rotate about X, Y, Z respectively, with positive sense given by the right-hand rule. Crucially, the standard specifies **relative motion of tool with respect to workpiece**, so axis letters describe tool motion even when it is physically the table that moves; primed letters denote the table-carrying-workpiece motions where they differ.

Mathematically, a WCS is a rigid transform \(T^{\text{MCS}}_{\text{WCS}}\in SE(3)\) mapping work coordinates to machine coordinates,

\[ \mathbf{p}_{\text{MCS}} = R\,\mathbf{p}_{\text{WCS}} + \mathbf{t}, \qquad R\in SO(3). \]

Controllers expose a bank of such offsets (commonly the G54 through G59 family) so several work zeros can coexist, one per fixture location on a tombstone or pallet. A tool-length offset and, on multi-axis machines, tool-center-point (TCP) or rotary-pivot compensation further compose with this transform so that programmed points refer to the intended cutting reference regardless of tool stickout or head articulation.

Most real parts need **multiple setups** because no single clamping exposes every face; a six-sided part might require three or more re-fixturings. Each setup carries its own WCS, its own safe/clearance planes, and its own set of accessible operations. Because re-fixturing reintroduces positioning error, datums and probing are used to re-establish the WCS consistently between setups; the choice of datum features (per the part's dimensioning scheme) directly governs how tolerances stack across setups. Good setup planning minimizes the number of clampings and aligns the WCS with the part's functional datums so that programmed dimensions match the drawing's.
