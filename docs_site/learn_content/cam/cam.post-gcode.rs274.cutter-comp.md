A milling cutter removes material along a swept envelope whose radius is finite, so the *center* of the tool never lies on the part boundary it is supposed to produce. **Cutter compensation** is the mechanism that reconciles the programmed path with the tool radius. There are two philosophies. In *computer* (or center-line) compensation the CAM system offsets the path by the tool radius when it generates the program, so the emitted \(X\,Y\) coordinates already describe the tool center. In *control* (or cutter) compensation the program describes the finished part profile and the machine controller offsets it at run time using the codes \(G41\) (compensation left of the direction of travel) and \(G42\) (right), with \(G40\) to cancel; the offset magnitude is read from a tool table indexed by a \(D\) word.

The governing idea is a signed offset of a planar curve. If the programmed contour is a curve \(\mathbf{p}(s)\) with unit tangent \(\mathbf{t}(s)\), the compensated tool-center path is
\[
\mathbf{c}(s) = \mathbf{p}(s) \pm r\,\mathbf{n}(s), \qquad \mathbf{n}(s) = \mathbf{t}(s)\times \hat{\mathbf{z}},
\]
where \(r\) is the tool radius, \(\mathbf{n}\) is the in-plane normal, and the sign is chosen by \(G41\) versus \(G42\). The compensation plane is fixed by the active plane select (\(G17/G18/G19\)). Control-side compensation lets a machinist compensate for tool wear or a reground cutter by editing a single offset value rather than reposting the program, which is the main reason it survives despite CAM systems being able to precompute offsets.

## Ramp-on, ramp-off, and offset singularities

Control compensation cannot be switched on instantaneously: the controller needs a *lead-in* move over which the tool transitions from the programmed point to the offset point, and a matching *lead-out* when cancelling. Turning compensation on or off is therefore only legal during a linear move, and the approach move must be at least as long as the tool radius or the controller faults. Offsetting also introduces geometric hazards. On a **concave** corner the two offset segments must be trimmed to their intersection, and if the corner radius is smaller than the tool radius the offset is infeasible, the tool physically cannot reach, producing a gouge or an alarm. On a **convex** (outside) corner the offset segments no longer meet, so the controller must insert a connecting arc or a bevel to keep the path continuous.

The reason cutter compensation lives partly in the controller and partly in the post-processor is division of concern. The CAM/post side must decide *which* scheme the target machine uses and emit consistent codes: with center-line compensation it bakes \(r\) into the coordinates and never emits \(G41/G42\); with control compensation it emits the part profile plus the correct \(G41/G42/D\) words and guarantees valid lead-in/lead-out geometry. A post that mixes the two, for example emitting already-offset coordinates *and* a \(G41\), doubles the offset and scraps the part. Correct handling of compensation direction, plane, lead moves, and corner cases is one of the most error-prone responsibilities a post-processor carries.
