Contouring (profiling) drives the tool along a single curve, open or closed, offset laterally by the tool radius so the finished wall coincides exactly with the programmed part edge. Unlike pocketing, which fills an area with many offsets, a contour is a single tool-radius offset of one boundary: the tool center travels on a path parallel to the part edge at distance \(R\) (plus any stock-to-leave). It is the workhorse for finishing outer perimeters, trimming to a profile, and cutting walls to size.

The side of the offset determines the milling direction relative to spindle rotation. Keeping the tool to one side gives climb milling (the chip starts thick and thins to zero), which generally yields better finish and tool life; the other side gives conventional milling. On a controller the offset can be applied in the part program as cutter radius compensation, so the programmer describes the actual part edge and the control shifts the path by a tool-diameter value from an offset register. This decouples the program from the exact cutter size and lets a worn or resharpened tool be compensated by editing one number, but it requires lead-in and lead-out moves for the compensation to activate cleanly.

## Depth passes, leads, and finishing
Wall height greater than a safe axial engagement is taken in several depth passes at the same XY contour, stepping down by \(a_p\) each level. To avoid a witness dwell mark where the tool first touches the wall, the path enters and exits on a tangent arc or angled lead rather than driving straight into the finished surface. A final spring pass at full depth with little or no radial stock removes deflection-induced error and produces the accurate, low-roughness wall. Because contouring commits the finished surface in a single offset, the tool-radius value, stock allowance, and any compensation offset must be exact, and the geometry must not contain internal radii smaller than the tool radius, which the tool physically cannot reach.
